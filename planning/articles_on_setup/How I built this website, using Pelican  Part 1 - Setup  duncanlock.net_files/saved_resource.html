                                        Duncan Lock Mod DardieD 7 months ago

                                        Thanks - glad you liked it!

                                        Yes, Pelican rebuilds everything each time.

                                        As far as I know, it's not currently possible to get Pelican to do incremental rebuilds.

                                        This probably hasn't been done because implementing it properly would be a bit difficult - what needs to be regenerated each time would depend completely on what you've modified. For example, if you change the theme, maybe everything or maybe just a css file, if you change a template, then maybe just the pages that use that template - but what about includes and template inheritance? If you've just added or modified a post, then that post page needs to be regenerated - but also all the index pages, category and tag pages, feeds, etc... that it might appear in - and it might have pushed other posts down in the pagination, so that would be all index pages and feeds.

                                        You would almost certainly have to do 90% of the work of a complete regeneration of everything - just to figure out what needs regenerating - and then regenerate those things. And you'd often end up regenerating much more than you think, even for 'simple' changes like a spelling correction in a post.

                                        Basically, it's probably not worth it.

                                        Having said that, I've been thinking about ways to speed the regeneration process up without changing Pelican and would like to have a play with ram disks at some point. If I get around to it, I'll write it up on this blog.

                                              Duncan Lock Mod Deny Dias 8 months ago

                                              Ah - you caught me out! This is a hack to my local version of Pelican that add's this piece of metadata. I added this after line 413 of generators.py:

                                              metadata['last_modified_date'] = datetime.datetime.fromtimestamp(os.stat(f).st_mtime)

                                              You can either do that, add a piece of metadata to each article with the same name, or take the reference to it out of the theme.

                                              My intention is to eventually finish this patch properly - which would be a more extensive change, making it behave the same way as article.date - and try and get it into upstream Pelican.
